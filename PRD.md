# 🍅 Pomodoro (番茄钟) Web 应用产品需求文档 (PRD)

## 1. 产品概述
本项目旨在构建一个纯前端实现的番茄钟 (Pomodoro) Web 应用程序。该应用无需任何复杂的后端服务器和数据库支持，所有数据直接保存在用户的浏览器本地。项目采用静态网站架构，最终将直接托管并部署在 **GitHub Pages** 上。

整体目标是提供一个轻量、极简、高颜值的专注力管理工具，帮助用户践行“25分钟工作 + 5分钟休息”的番茄工作法。

## 2. 目标用户
- 需要提高专注度的开发者、学生与知识工作者。
- 偏好轻量级应用、不希望进行繁琐注册登录流程的用户。

## 3. 核心功能模块 (MVP)

### 3.1 核心计时器 (Timer)
- **三种模式无缝切换：**
  - **专注模式 (Pomodoro):** 默认 25 分钟。
  - **短休息 (Short Break):** 默认 5 分钟。
  - **长休息 (Long Break):** 默认 15 分钟（建议在完成 4 个 Pomodoro 后进行）。
- **流程控制：**
  - 开始 (Start)、暂停 (Pause)、重置 (Reset)。
- **视觉与系统级反馈：**
  - 页面中央通过超大字体展示倒计时（格式 MM:SS）。
  - **浏览器 Tab 标题映射：** 实时在网页标签上显示倒计时，方便用户在浏览其他网页时掌握时间。
  - **动态主题配色：** 根据当前模式自动切换页面主色调（如专注时用沉浸红/深色，休息时用放松绿/蓝色）。

### 3.2 简易任务清单 (Task List)
- **任务增删改查：** 添加新任务、编辑任务内容、删除不需要的任务。
- **状态管理：** 标记任务为“已完成”，完成的任务文本加上删除线。
- **专注焦点：** 允许用户从任务列表中选中一个作为“当前正在专注的任务”，并在计时器显眼位置进行展示。

### 3.3 个性化设置 (Settings)
- **自定义时长：** 允许用户修改番茄钟、短休息和长休息的默认分钟数。
- **声音提醒：** 提供倒计时结束时的全局提示音（铃声），并附带静音/开启开关。

### 3.4 本地数据持久化 (Local Storage)
- 无需账号体系。利用 HTML5 的 `localStorage` 技术，自动保存以下数据：
  - 用户自定义的时长与设置偏好。
  - 任务列表的当前内容及完成状态。
- 确保用户下次打开网站或者刷新浏览器时，体验是连贯的、数据不丢失。

## 4. 非功能性需求

- **零后端依赖：** “仅使用前端”是最高原则，不可引入任何动态后端 API。
- **响应式 UI 设计 (Responsive Design)：** 完美适配 PC 屏幕和平板/手机等移动设备屏幕。
- **高品质交互：** 添加平滑的 CSS 过渡动画（如颜色切换、按钮悬浮、弹窗出现等），提升应用的高级感。
- **自动化部署：** 配置 GitHub Actions 工作流，实现代码推送到主分支后自动构建并部署至 GitHub Pages。

## 5. 建议技术栈
考虑到项目的无后端性质及现代前端构建体验，推荐以下技术选型：
- **核心框架：** Vue 3 (Composition API) 或纯原生 Vanilla JS (如果追求极简)。考虑到组件化维护，Vue 3 是极佳选择。
- **构建工具：** Vite。能极速启动本地服务且打包出的静态产物非常小。
- **样式方案：** Vanilla CSS / SCSS，搭配现代 CSS 变量实现动态主题切换；或使用 Tailwind CSS 快速构建原子化样式。
- **部署基建：** GitHub Pages 提供静态托管服务。

## 6. 开发与发布节奏
- **第一阶段：** 搭建 Vite 工程，完成 HTML/CSS 基础骨架，实现倒计时核心逻辑与主题变色。
- **第二阶段：** 整合任务列表 (Todo List) 逻辑，串联选中任务与计时器。
- **第三阶段：** 接入 LocalStorage 持久化，完成自选设置和提示音功能。
- **第四阶段：** 编写 GitHub Actions 工作流 `.yml`，完成向 GitHub Pages 的上线与可用性测试。
